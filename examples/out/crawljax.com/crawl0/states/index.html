      <h1 class="entry-title"><a href="/blog/2014/03/07/crawljax-gets-a-face-lift-web-ui-in-crawljax-3-dot-5/">Crawljax Gets a Face Lift. Web UI in 3.5</a></h1>
    
    
      <p class="meta">
        








  


<time datetime="2014-03-07T11:36:00+01:00" pubdate="" data-updated="true">Mar 7<span>th</span>, 2014</time>
        
      </p>
    
  </header>


  <div class="entry-content"><p>We just released Crawljax 3.5 and with it comes a web interface for Crawljax. It supports all the major functionality of crawljax such as adding/editing crawl configurations, installing plugins, viewing crawl logs in real time and displaying plugin output. Kudos to the developers, <a href="https://github.com/jeremyhewett">@jeremyhewett</a> and <a href="https://github.com/ryansolid">@ryansolid</a>.
To run it, simply unzip the web distribution archive and run the command <code>java -jar crawljax-web-3.5-SNAPSHOT.jar</code>. You can customize the port and output directory using <code>-p</code> and <code>-o</code> arguments. Here are some screen-shots of the web interface in action. More usage tutorials will follow.</p>

<p><img src="/images/webui1.jpg" title="&quot;Crawljax Web UI Home&quot;" alt="&quot;Crawljax Web UI Home&quot;">
Crawljax Web UI Home – You can start by adding a ‘New Configuration’ as shown</p>

<p><img src="/images/webui2.jpg" title="&quot;Edit Configuration Page&quot;" alt="&quot;Edit Configuration Page&quot;">
‘Edit Configuration’ Page</p>

<p><img src="/images/webui3.jpg" title="&quot;Add Plugin Page&quot;" alt="&quot;Add Plugin Page&quot;">
‘Add Plugin’ Page</p>

<p><img src="/images/webui4.jpg" title="&quot;Crawl Status Log&quot;" alt="&quot;Crawl Status Log&quot;">
Crawl status log with real-time updates</p>

<p><img src="/images/webui5.jpg" title="&quot;Crawl Overview Output&quot;" alt="&quot;Crawl Overview Output&quot;">
Output of the Crawl Overview plugin</p>

<p>Also in release 3.5:</p>

<ul>
<li>Deprecated the malfunctioning DomChangeNotifierPlugin. Introuced StateVertexFactoryl. <a href="https://github.com/crawljax/crawljax/issues/347">#347</a></li>
<li>Better PhantomJS support. Tests run on PhantomJS by default.</li>
<li>Switched from URL’s to URI’s for better performance. <a href="https://github.com/crawljax/crawljax/issues/322">#322</a></li>
</ul>


<p><a href="https://github.com/crawljax/crawljax/releases">You can download the release from our GitHub project site.</a></p>

      <h1 class="entry-title"><a href="/blog/2013/05/05/release-3-dot-0-is-out/">Release 3.0 Is Out</a></h1>
    
    
      <p class="meta">
        








  


<time datetime="2013-05-05T17:14:00+02:00" pubdate="" data-updated="true">May 5<span>th</span>, 2013</time>
        
      </p>
    
  </header>


  <div class="entry-content"><p>Release 3.0 is out. This is a major release, which contains many key updates and renovations. The release contains <a href="https://github.com/crawljax/crawljax/issues?milestone=3&amp;state=closed">several bug fixes and loads of enhancements</a>. The code base has been split up into modules, the API has changed a little, and the crawl overview plugin has been completely renovated.</p>

<h3>New overview plug-in</h3>

<p>Most notable in the new release is the new overview plugin. The plugin shows an interactive state graph of the crawl and some statistics. Make sure you check out <a href="http://crawls.crawljax.com">a demo</a> or try one of our <a href="https://github.com/crawljax/crawljax/tree/master/examples">examples</a> yourself.</p>

<p><img src="/images/new-overview-plugin.png" title="&quot;The new overview plugin&quot;" alt="&quot;The new overview plugin&quot;"></p>

<h3>New command line interface</h3>

<p>This release brings more support for command line configuration. Once you download the zip, running Crawljax can simply be done using the command:</p>

<pre><code>java -jar crawljax-cli-version.jar http://your.site.com outputfolder
</code></pre>

<p>Crawljax will Crawl that site with the new Crawl overview plugin enabled. You can run <code>java -jar crawljax-cli-version.jar</code> to see a list of possible configurations for the crawl.</p>

<p>The zip is downloadable from <a href="http://search.maven.org/#search%7Cga%7C1%7Cg%3A%22com.crawljax%22%20AND%20a%3A%22crawljax-cli%22">the central Maven repository</a>.</p>

<h3>Other important updates</h3>

<ul>
<li>Crawljax is now configured using a configuration builder. You start your configuration using <code>CrawljaxConfiguration.builderFor("http://your.website.com");</code>.</li>
<li>The project has been split up in three versions: <em>core</em>, <em>cli</em> and <em>examples</em>. The <em>cli</em> modules contains the command line interface. The <em>core</em> module can be included in any project as a jar to run Crawljax programmatically. The <em>examples</em> module is the easiest way to try out several configurations of Crawljax in your favorite IDE. Check out our updated documentation for more details.</li>
<li>You can configure the crawler to crawl all found <code>href</code> attributes. Even if the elements are not visible because they only show up when the crawler hovers on another element.</li>
<li>You can now configure the crawler so that it does not click any children of a certain element using a short syntax like <code>dontClickChildrenOf("LI").withId("dontClickMe");</code></li>
<li>Major performance and stability improvements.</li>
</ul>


<p>You can <a href="https://github.com/crawljax/crawljax/issues?milestone=3&amp;page=1&amp;state=closed">view all closed issues</a> or <a href="https://github.com/crawljax/crawljax/compare/crawljax-2.2...crawljax-3.0">the full diff</a> on Github.</p>
</div>
